# Desarrollo de Aplicaciones WEB

Estructura del repositorio personal para entregas

Cada estudiante deberá crear un repositorio personal en GitHub que será
utilizado durante todo el curso para la entrega de tareas, prácticas y
actividades realizadas en clase.

El objetivo es mantener un orden claro, cronológico y profesional del
trabajo realizado durante el semestre, similar a como se gestionan
proyectos en entornos reales de desarrollo de software.

  -----------------------------------------
  1. ORGANIZACIÓN GENERAL DEL REPOSITORIO
  -----------------------------------------

Dentro del repositorio se deberán crear carpetas por unidad temática
utilizando la siguiente estructura:

-  /UNIDAD1 
-  /UNIDAD2 
-  /UNIDAD3

Cada carpeta representará los contenidos y actividades correspondientes
a esa unidad del curso.

  ------------------------------------------------------
  2. ORGANIZACIÓN DE ACTIVIDADES DENTRO DE CADA UNIDAD
  ------------------------------------------------------

Dentro de cada carpeta de unidad se crearán subcarpetas para cada
actividad:

-   Tareas
-   Prácticas
-   Actividades de clase

El nombre de cada carpeta deberá seguir la siguiente estructura:

MMDDTIPONUMERO

Donde:

  -  MM = Mes a dos dígitos
  -  DD = Día a dos dígitos
  -  TIPO = Tipo de actividad (TAREA, PRACTICA, CLASE)
  -  NUMERO = Número de la actividad a dos dígitos

  ------------------------------------
  3. EJEMPLOS DE NOMBRES DE CARPETAS
  ------------------------------------

  -  0401TAREA01
  -  0403CLASE01
  -  0408PRACTICA01
  -  0415TAREA02
  -  0420PRACTICA02

Este formato permite identificar fácilmente: 

  - La fecha en que se asignó
  - La actividad
  - El tipo de actividad
  - El orden de las entregas.


  ------------------------------------
  4. CONTENIDO DENTRO DE CADA CARPETA
  ------------------------------------
  
Cada carpeta de actividad debe contener:

-   Archivos de código desarrollados
-   Recursos utilizados en la actividad
-   Un archivo README.md explicando brevemente el trabajo realizado

Ejemplo:

-  /UNIDAD1/0401TAREA1/index.html
-  /UNIDAD1/0401TAREA1/estilos.css
-  /UNIDAD1/0401TAREA1/README.md

  ----------------------------------------------
  5. CONTENIDO DEL README.md DE CADA ACTIVIDAD
  ----------------------------------------------

Cada carpeta debe incluir un archivo README.md con la siguiente
información mínima:

-  Nombre de la Actividad
-  Alumno
-  Fecha
-  Unidad
-  Descripción: Breve explicación de la actividad realizada.
-  Objetivo: Qué se buscaba aprender o desarrollar con la actividad.
-  Archivos incluidos: Lista de archivos incluidos en la carpeta.
-  Notas: Comentarios adicionales si son necesarios.

  -----------------------
  6. REGLAS IMPORTANTES
  -----------------------

1.  Todas las entregas deberán subirse exclusivamente a GitHub.
2.  Las carpetas deberán respetar exactamente el formato de nombres
    indicado.
3.  Cada actividad deberá colocarse dentro de su unidad correspondiente.
4.  No se aceptarán archivos sueltos fuera de la estructura definida.
5.  El repositorio debe mantenerse actualizado durante todo el semestre.
6.  Cada actividad debe incluir su README.md correspondiente.
7.  Por ningún motivo se deben subir cambios o archivos a este repositorio.

  -------------------------------------
  7. README PRINCIPAL DEL REPOSITORIO
  -------------------------------------

El repositorio deberá incluir un README.md principal con la siguiente
información:

-  Alumno
-  Asignatura
-  Profesor
-  Carrera Semestre
-  Descripción: Repositorio que contiene todas las actividades, prácticas y tareas realizadas durante el curso.
-  Estructura del repositorio: Las actividades se encuentran organizadas por unidades y por fecha de asignación.

  ----------------------------------
  9. BUENAS PRÁCTICAS RECOMENDADAS
  ----------------------------------

Se recomienda a los estudiantes:

-   Realizar commits frecuentes durante el desarrollo de las
    actividades.
-   Escribir mensajes de commit claros y descriptivos.
-   Mantener el repositorio ordenado y actualizado.
-   Utilizar GitHub como portafolio de evidencias de aprendizaje.

Ejemplo de mensajes de commit:

  - Se agrega práctica 1 de HTML
  - Se corrige estructura de la tarea 2
  - Se agregan estilos CSS a la práctica
